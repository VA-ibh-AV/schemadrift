The Problem

Producers change schemas.
Consumers find out the hard way.

In microservice architectures, a producer can silently rename a field, change a float64 to a string, or add an undocumented key — and your consumer receives no error at the broker level. The message is delivered. Your service just behaves wrongly.

✓  Before — what your consumer expects

{
  "alert_id": "abc-123",
  "value":    87.5,            ← float64
  "fired_at": "2025-05-08T10:00:00Z",  ← RFC3339 string
  "resolved": false
}

✗  After — what the producer silently changed

{
  "alert_id":  "abc-123",
  "value":     "87.5%",        ← DRIFT: number → string
  "fired_at":  1715161200,     ← DRIFT: string → number
  "severity":  "critical"      ← DRIFT: new field
  // "resolved" removed          ← DRIFT: missing field
}
Features

Everything you need.
Nothing you don't.

Drop one library into your consumer. No config files, no schema definitions, no external services.

🧠

Self-learning baseline

Observes N messages during warm-up to infer field types, required-ness, and nullability. No schema to write by hand.

⚡

First-message detection

Catches drift on the very first message that doesn't match the learned baseline — not after thousands of bad messages.

🔒

Three enforcement policies

Warn and continue, block and return an error, or quarantine to a dead-letter queue. Choose per-topic or per-stream.

💾

Baseline persistence

Atomically saves the learned baseline to disk. Restarts load instantly — no re-learning from scratch.

📦

Zero core dependencies

pkg/drift uses only stdlib. Adapters depend solely on the broker SDK you already use.

📊

Pluggable metrics

Implement metrics.Recorder to emit to OTel, Prometheus, StatsD, or any backend.

Fingerprinter

One JSON decode pass. Produces a flat map[fieldPath]FieldType using dot-notation and items[].id for arrays.

Baseline (learning → frozen)

Accumulates N samples, then atomically freezes into a lock-free snapshot. Post-freeze reads need no mutex.

Detector → Enforcer

Stateless diff against frozen baseline. Four event kinds. Enforcer applies Warn / Block / Quarantine.

Examples

Up and running
in minutes.

Works with any broker via the generic adapter, or drop-in interceptors for Kafka and Redis Streams.

consumer.go 
import (
    "github.com/VA-ibh-AV/schemadrift/adapters/generic"
    "github.com/VA-ibh-AV/schemadrift/pkg/drift"
)

interceptor, err := generic.New(generic.Config{
    Source: "alerts.events.firing",
    Baseline: drift.BaselineConfig{
        LearningSamples:     100,
        AllowOptionalFields: true,
        AllowNullable:       true,
    },
    Policy: drift.PolicyConfig{
        Policy: drift.PolicyWarn,
        OnDrift: func(report drift.DriftReport) {
            log.Printf("drift on %s: %d event(s)", report.Source, len(report.Events))
        },
    },
    StorePath: "/var/lib/myservice/alerts-baseline.json",
}, func(ctx context.Context, payload []byte) error {
    return processAlert(payload) // your business logic
})

// In your consume loop:
for msg := range messages {
    if err := interceptor.Handle(ctx, msg.Value); err != nil {
        log.Error("drift enforcement rejected message", "err", err)
    }
}
kafka_consumer.go 
import (
    "github.com/VA-ibh-AV/schemadrift/adapters/kafka"
    "github.com/VA-ibh-AV/schemadrift/pkg/drift"
)

interceptor, err := kafka.NewInterceptor(kafka.ConsumerConfig{
    Baseline: drift.BaselineConfig{
        LearningSamples:     100,
        AllowOptionalFields: false, // strict: every field required
        AllowNullable:       false,
    },
    Policy: drift.PolicyConfig{
        Policy: drift.PolicyQuarantine,
        DLQ:    myDLQSink,
        OnDrift: func(report drift.DriftReport) {
            for _, ev := range report.Events {
                log.Printf("[%s] field=%s expected=%s got=%s",
                    ev.Kind, ev.Field, ev.Expected, ev.Got)
            }
        },
    },
    StoreDir: "/var/lib/myservice/baselines", // one file per topic
})

for {
    msg := reader.ReadMessage(ctx)
    err := interceptor.Handle(ctx, msg, func(ctx context.Context, m kafkago.Message) error {
        return processMessage(m.Value)
    })
    if err != nil {
        log.Error("blocked or quarantined", "err", err)
    }
}
redis_consumer.go 
import (
    "github.com/VA-ibh-AV/schemadrift/adapters/redis"
    "github.com/VA-ibh-AV/schemadrift/pkg/drift"
)

interceptor, err := redis.NewInterceptor(redis.ConsumerConfig{
    Baseline: drift.BaselineConfig{LearningSamples: 50},
    Policy:   drift.PolicyConfig{Policy: drift.PolicyWarn},
    StoreDir:     "/var/lib/myservice/baselines",
    PayloadField: "data", // JSON is in this XMessage field
})

// XRead loop — streams: [name1, name2, id1, id2]
streams, _ := rdb.XRead(ctx, &goredis.XReadArgs{
    Streams: []string{"orders:updates", "users:events", lastID["orders:updates"], lastID["users:events"]},

for _, s := range streams {
    for _, msg := range s.Messages {
        interceptor.Handle(ctx, s.Stream, msg, func(ctx context.Context, m goredis.XMessage) error {
            return processOrder(m)
        })
    }
}
Enforcement Policies

You choose what
happens on drift.

All policies emit the OnDrift callback before enforcement — hook in your metrics, alerts, or logging there.

PolicyWarn
Logs the drift report and lets the message through to your handler. Drift is visible but non-blocking.
Use when: You want observability without risk. Early adoption, canary rollout, high-optionality schemas.
PolicyBlock
Returns an error. The message never reaches your handler. The caller decides whether to retry, skip, or page.
Use when: Strict enforcement. Downstream systems can't tolerate malformed input. Critical financial or safety pipelines.
PolicyQuarantine
Writes the original payload + full drift report to a DLQ sink, then returns an error. Drifted messages are replayable.
Use when: You want enforcement now but need the ability to replay after a producer-side schema upgrade is confirmed.
Drift Events

Four kinds of drift.
All covered.

Every DriftReport carries one or more events with the field path, expected value, and actual value.

type_change

Type Change

A field's JSON wire type changed between the baseline and the incoming message. e.g. value was number, received as string.

missing_field

Missing Field

A field marked required in the baseline is absent from the incoming message. e.g. host was always present during learning but is now gone.

new_field

New Field

A field appears in the message that was never seen during the learning phase. e.g. producer added severity without notice.

nullability_change

Nullability Change

A field that was never null during learning arrives as null. Only fires when AllowNullable: false.
